Lecture 2: Introduction to Web Applications

Introduction

The World Wide Web (WWW) has evolved from simple static pages to complex web-based applications that allow real-time interaction, collaboration, and communication.

  • Early web pages were static and content-driven.
  • Modern websites are dynamic and interactive.
  • Web technologies have enabled global communication, business, and learning.

Static page example:

<!DOCTYPE html> <html> <body> <h1>Hello World</h1> </body> </html>

Dynamic example (JS):

<script> document.write('Time: ' + new Date()); </script>

Categories of Web Applications

  • Static Web Applications: Simple content-based websites with fixed information (e.g., portfolios, company pages).
  • Dynamic Web Applications: Data-driven sites that change based on user interaction (e.g., social media, e-commerce).
  • E-commerce Applications: Allow users to buy and sell goods online (e.g., Amazon, Daraz).
  • Portal Web Applications: Provide multiple services from a single interface (e.g., Yahoo, MSN).
  • Content Management Systems (CMS): Enable non-technical users to manage content (e.g., WordPress).

Example form snippet:

<form> <input type="text" name="name" placeholder="Your Name"> <button type="submit">Send</button> </form>

Document-Centric Web Applications

Document-centric applications focus on creating, managing, and sharing documents through a web interface.

  • Examples include Google Docs, Office 365, and Dropbox Paper.
  • Users can create, edit, and collaborate in real-time.
  • They often support cloud storage and version control.

Key Benefit: Access your documents from anywhere using an internet connection.
